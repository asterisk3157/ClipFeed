<h1 align="center">ClipFeed</h1>

<p align="center"><em>コピーした内容を、ワンクリックで Markdown / HTML / PDF / 画像のファイルにして保存する macOS メニューバーアプリ。</em></p>

---

Web などでコピーした内容を、メニューから形式を選ぶだけで **ファイル**として `~/Downloads` に保存し、
そのファイルをそのままクリップボードに載せ直します。貼り付け先で ⌘V するだけで、
AI への添付やチャットへの貼り付けがすぐにできます。

## どれを使う？（用途別）

| こんなとき | 選ぶメニュー | できあがるファイル |
|---|---|---|
| 🤖 AI に文章を渡したい・トークンを節約したい | **.md に変換** | 画像を除いた軽量な Markdown |
| 🧩 表やリンクなど構造を丸ごと残したい | **.html で保存** | 元の HTML |
| 📄 見た目ごと1枚で残したい・印刷や共有に | **1枚の PDF で保存** | 文字くっきりの高精細 PDF（軽い・拡大に強い） |
| 🖼 チャットや SNS にそのまま貼りたい | **1枚の画像 (PNG) で保存** | 高解像度の画像 |

> Markdown は画像を除くので軽量（実測で約43%減）。
> 長いページを「画像」で残すと縦に長く重くなりがちなので、その場合は **PDF がおすすめ**です。

## 使い方

1. Web ページなどで内容をコピー（⌘C）
2. メニューバーの 📄 アイコンをクリックし、上の表を参考に保存したい形式を選ぶ
3. `~/Downloads` にファイルが保存され、クリップボードがそのファイルに切り替わる
4. 貼り付けたい場所で ⌘V
   - **Claude デスクトップアプリ** → ファイルが添付される
   - **ターミナル（Claude Code 等）** → ファイルの絶対パスが入る

## ログイン時に自動起動

メニューの **「ログイン時に起動」** をクリックするだけで ON / OFF できます。
（安定動作のため、`/Applications` に置いてから ON にするのがおすすめ。macOS 13 以降が必要）

## インストール

追加インストール不要の自己完結アプリです。ビルドには Xcode Command Line Tools（`swiftc`）が必要です。

```bash
git clone https://github.com/asterisk3157/ClipFeed.git
```

```bash
cd ClipFeed && ./build.sh
```

生成された `ClipFeed.app` を `/Applications` に置いて常用できます（リポジトリ直下で）:

```bash
cp -R ClipFeed.app /Applications/
```

## 必要環境

- macOS 13 以降

## 既知の注意点

- 初回実行時、`~/Downloads` へのアクセス許可ダイアログが出ることがあります（許可で OK）
- 貼り付け先がファイル添付に対応していない欄では、代わりに絶対パスの文字列が貼られます
- PDF / 画像は、コピー元サイトのデザイン CSS までは再現しません（読みやすく整えた見た目で保存されます）。サイトの動的なカルーセル等は一部再現できない場合があります
- ローカルビルド（アドホック署名）のため、配布された `.app` の初回起動時に Gatekeeper の確認が出る場合があります
